# AGENTS.md - nisli

## Shape

- This repo publishes `@nisli/core` from `packages/core` and `@nisli/ssg` from `packages/ssg`.
- Root `package.json` is private and only delegates workspace scripts.
- npm package README is `packages/core/README.md`.
- Root `README.md` should carry the same user-facing framework information for GitHub.
- Deep framework authoring rules live in `.agents/skills/nisli-framework`.

## Commands

- `pnpm install`
- `pnpm build`
- `pnpm test`
- `pnpm typecheck`

## Framework Work

- Use `.agents/skills/nisli-framework` when writing, reviewing, or debugging Nisli components.
- Tests live beside source in `packages/core/src`.
- Vitest runs with `happy-dom`.
- Prefer testing public framework APIs: `signal`, `component`, `html`, `inject`, `query`, etc.
- Keep `index.ts` files as barrel exports only.
- Do not hand-edit `dist`; it is generated by `pnpm build`.

## Package Architecture

- Dev exports point at TypeScript source.
- Published exports point at built files via `publishConfig`.
- Keep `@nisli/core` focused on component authoring and the browser runtime.
- Keep static rendering and SSG build tooling in `@nisli/ssg`.

## Publishing Invariants

- Workflow file is `.github/workflows/auto-tag.yml`.
- npm Trusted Publisher must be configured with workflow filename `auto-tag.yml`.
- The workflow intentionally mirrors `backlog-mcp`'s proven `auto-tag.yml` pattern.
- Keep `node-version: '24'`.
- Keep `npm install -g npm@latest`.
- Use `npm publish`, not `pnpm publish`, for OIDC trusted publishing.
- Keep package repository metadata pointed at this repo:
  - `https://github.com/gkoreli/nisli.git`
  - directory `packages/core` for `@nisli/core`
  - directory `packages/ssg` for `@nisli/ssg`

## Publishing Failure Pattern

- npm may return a misleading `404 Not Found - PUT https://registry.npmjs.org/@nisli%2fcore`.
- That usually means trusted-publisher identity mismatch, not that the package is missing.
- Check workflow filename, package repository metadata, Node/npm version, and npm Trusted Publisher settings.
- The workflow creates the GitHub tag/release before npm publish.
- If publish fails after tag creation, delete the failed release/tag before retrying the same version:
  - `gh release delete vX.Y.Z --cleanup-tag --yes`

## Release Checklist

- Bump `packages/core/package.json`.
- Confirm the version is free:
  - `npm view @nisli/core@X.Y.Z version`
  - `E404 No match found for version` means it is available.
- Run `pnpm build` and `pnpm test`.
- Push to `main`.
- Watch the run:
  - `gh run list --workflow auto-tag.yml --limit 5`
  - `gh run watch <run-id> --exit-status`
- Confirm npm:
  - `npm view @nisli/core version repository --json`
